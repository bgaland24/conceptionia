# RG-005 — Habilitation à organiser le pouvoir de décision

!!! warning "Statut : À valider"

## Énoncé

La **Directrice de la DQ** est la seule personne habilitée à **organiser le pouvoir de décision** pour les activités de contrôle. Elle peut déléguer ce pouvoir à des délégataires désignés, mais cette délégation est formelle et tracée.

## Portée

- **Processus :** [PR-01 — Cycle de contrôle officiel](../processus/cycle-controle.md)
- **Étape :** Décision (étape 5)

## Contraintes techniques

```sql
-- Table: habilitation_decision
-- Modélise les délégations de pouvoir de décision accordées par la Directrice DQ
CREATE TABLE habilitation_decision (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    id_utilisateur UUID NOT NULL REFERENCES utilisateur(id),
    type_activite VARCHAR(100),         -- périmètre de la délégation
    date_debut DATE NOT NULL,
    date_fin DATE,                       -- NULL = indéterminée
    accordee_par UUID NOT NULL REFERENCES utilisateur(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT now(),
    updated_at TIMESTAMP WITH TIME ZONE,
    CONSTRAINT ck_habilitation_dates CHECK (date_fin IS NULL OR date_fin > date_debut)
);
```

## Points à clarifier

!!! question "À valider avec les sachants"
    - Les délégations sont-elles par type d'activité de contrôle, par région, ou globales ?
    - Une délégation peut-elle être temporaire (ex : absence de la Directrice) ?
    - Y a-t-il des décisions qui ne peuvent jamais être déléguées ?
