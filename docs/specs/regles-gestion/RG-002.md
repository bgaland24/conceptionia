# RG-002 — Programmation ajustée au fil de l'eau

!!! warning "Statut : À valider"

## Énoncé

En cours d'année, la programmation initiale est **réajustée** pour intégrer les demandes et déclarations ad hoc des opérateurs. Cette programmation ajustée est **dynamique** et peut évoluer tout au long de l'année.

## Portée

- **Processus :** [PR-01 — Cycle de contrôle officiel](../processus/cycle-controle.md)
- **Étape :** Programmation (étape 2b)

## Contraintes techniques

```sql
-- La programmation ajustée est une version complémentaire, jamais un remplacement de l'initiale
-- type_programmation IN ('initiale', 'ajustee')
-- Une entrée de type 'ajustee' référence obligatoirement une programmation initiale parente
ALTER TABLE programmation
    ADD CONSTRAINT ck_programmation_ajustee_parente
    CHECK (type_programmation = 'initiale' OR id_programmation_initiale IS NOT NULL);
```

## Points à clarifier

!!! question "À valider avec les sachants"
    - La programmation ajustée remplace-t-elle ou complète-t-elle la programmation initiale en base ?
    - Y a-t-il une validation formelle de chaque ajustement ou est-ce continu ?
